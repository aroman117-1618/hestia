## Key Learnings

1. **Long-running sessions ≠ stuck sessions.** The 1h24m timer is cumulative session duration, not current-command duration. The actual running command (pytest) was only 27s in — well within normal range for 977 tests. The UI's `Running... (Ns)` timer is the reliable indicator.

2. **Ctrl+B (background task) has a known tmux conflict.** Tmux uses Ctrl+B as its prefix key, silently consuming the first press. Users in tmux must press Ctrl+B twice. No `~/.claude/keybindings.json` existed, confirming defaults were active.

3. **`/doctor` dismissed with no output 3 times.** Either the diagnostics found nothing (healthy config) or the UI dismissed too quickly to read. The lack of persistent output from `/doctor` is a UX gap — there's no way to review what it checked.
