## Key Learnings

1. **pytest hangs after test completion.** ChromaDB spawns background threads that prevent process exit. The process reports all tests passed but never terminates. This affects any script that runs `python -m pytest` and waits for exit. Workaround: background the process, set a watchdog timer, kill after timeout, check captured output for pass/fail.
2. **macOS has no `timeout` command.** Neither `timeout` nor `gtimeout` are available by default — requires `brew install coreutils`. The portable alternative is a background-process + watchdog pattern in bash.
3. **xcodebuild daemons inherit file descriptors.** When piping xcodebuild output (e.g., `2>&1`), spawned daemons can keep the pipe open after the build completes. Redirect to a temp file instead of piping to avoid hangs.
4. **`set -euo pipefail` + manual exit code checking don't mix.** Commands whose exit codes you want to inspect need `set +e` bracketing, otherwise `set -e` exits the script before `$?` can be captured.
5. **xcodebuild requires `-project` flag from repo root.** Running `xcodebuild -scheme HestiaWorkspace` from the repo root fails (no project found). Must specify `-project HestiaApp/HestiaApp.xcodeproj`.
6. **Plan audit conditions prevented a wasted hook.** The /plan-audit correctly identified that installing the hook without fixing health tests first would lead to `--no-verify` muscle memory. Fixing those tests as step 0 was the right call.

## Root Cause of Low Efficiency

The pytest hang was a genuinely novel discovery — it's not something that would appear in normal test runs (the Bash tool seems to handle it differently than a direct invocation, and prior sessions never ran pytest inside a shell script wrapper). The debugging spiral could have been cut to ~4 turns with this approach:

1. Test pytest in isolation first: `python -m pytest tests/ -q && echo DONE` (would hang → identifies pytest as the problem)
2. Test with subprocess timeout (would confirm the process never exits)
3. Implement `run_with_timeout` directly

Instead, the session investigated the script wrapper, xcodebuild interactions, pipe buffering, and sandbox behavior before isolating the root cause. **Lesson: when a script hangs, bisect by testing each command in isolation before debugging the script itself.**
