## Key Learnings

1. **Agent output extraction is fragile.** Sub-agent output files use JSON streaming format (one JSON object per line), not plain text. The backend explorer produced 74K tokens of streaming JSON, and the final assistant message couldn't be extracted cleanly because it wasn't generated (agent hit turn limit before synthesizing). The other 3 agents successfully returned their results via the task notification system.

2. **pytest hangs without timeout config.** Running `python -m pytest tests/ -q --tb=no` hung indefinitely — likely ChromaDB or Ollama connection attempts blocking. This is both a session friction point AND a legitimate audit finding. The `--timeout=10` flag also hung, suggesting pytest-timeout may not interrupt all blocking operations (e.g., native socket connects).

3. **4 parallel explorers is effective but needs supervision.** Security, tests/docs, and iOS explorers all produced excellent comprehensive reports. Backend explorer ran out of budget doing individual greps without synthesizing. The pattern of "launch 4 in parallel, then manually fill gaps" worked well overall.

4. **`detail=str(e)` is the most common security violation.** Found in 5 route locations (wiki.py, agents_v2.py, tasks.py). This pattern leaks internal error structure to HTTP clients. The convention (`sanitize_for_log` + generic detail) is well-documented but not enforced by automation.

5. **Codebase is genuinely healthy.** Zero upward layer imports, zero bare excepts, zero force-unwraps in Swift, zero hardcoded secrets. The architecture decisions from 15+ sessions ago continue to hold up well.
