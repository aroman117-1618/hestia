## Key Learnings

1. **Context continuations carry stale assumptions.** The conversation summary described code with bad imports that had already been partially fixed between sessions. Always re-read files from disk before "fixing" something described in a summary.
2. **APIClient `private` methods are a recurring friction point.** Every new ViewModel that needs HTTP calls has to work around the fact that `get`/`put`/`delete` were private. Making them `internal` was the right call — enables direct generic HTTP calls without boilerplate protocol methods.
3. **Logger initialization pattern is not in CLAUDE.md.** The project uses `get_logger()` (no arguments), not `HestiaLogger(component=...)`. This pattern needs to be documented to prevent repeat errors.
4. **macOS app follows clean patterns.** The `navIcon()` + `WorkspaceView` enum + NotificationCenter routing pattern made wiring new views trivial once the file structure was understood.
