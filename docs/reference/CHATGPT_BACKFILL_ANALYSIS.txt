BACKFILL STRATEGY FOR HESTIA MEMORY SYSTEM
================================================================================

PHASE 1: EXTRACTION & STRUCTURE (1-2 hours)
──────────────────────────────────────────

1. Parse all conversation files and extract:
   - Conversation title, creation date, message count
   - First and last message (frame context)
   - Full conversation tree (all messages)
   - Message roles (user vs. assistant)
   - Timestamp for each message (temporal context)

2. Create JSON metadata file per conversation:
   {
     "id": "conversation_uuid",
     "title": "string",
     "create_date": "YYYY-MM-DD",
     "create_timestamp_unix": integer,
     "message_count": integer,
     "category": "Project-specific|Technical|Professional|Personal|Creative|Research|Transactional",
     "memory_value": "HIGH|MEDIUM|LOW",
     "messages": [
       {
         "timestamp": unix_timestamp,
         "role": "user|assistant",
         "content": "string",
         "summary": "brief summary if >500 chars"
       }
     ],
     "key_themes": ["theme1", "theme2", ...],
     "importance_score": 0.0-1.0
   }

3. Create aggregated index file:
   - Links conversations to related Hestia modules/features
   - Cross-reference to SPRINT.md and docs/
   - Temporal clustering (group by project phase)

PHASE 2: ENRICHMENT & TAGGING (2-3 hours)
──────────────────────────────────────────

1. Tag each conversation with:
   - Project phase (Trading Sprint 21-30, etc.)
   - Feature area (Council, Memory, Trading, Health, etc.)
   - Decision type (ARCHITECTURE, STRATEGY, DEBUGGING, LEARNING, etc.)
   - Priority level (1-3)
   - Related ADRs (if applicable)
   - Related documentation files

2. Extract key decisions/insights:
   - For each project-specific conversation:
     * What decision was made? Why?
     * What constraints exist?
     * What alternatives were considered?
     * What's the expected outcome?
   - For personal preference conversations:
     * What's important to Andrew?
     * What's his communication style?
     * What are his goals?

3. Create summaries:
   - 1-2 paragraph executive summary
   - Key quotes/decisions
   - Recommended import priority

PHASE 3: INTEGRATION INTO HESTIA MEMORY (3-4 hours)
────────────────────────────────────────────────────

1. Import into ChromaDB:
   - Create embeddings for each conversation
   - Store full conversation text + metadata
   - Enable semantic search ("tell me when Andrew discussed...")
   - Set similarity threshold for retrieval

2. Import into SQLite:
   - Store facts table: temporal facts about decisions/context
   - Link to conversation ID for audit trail
   - Set creation dates to preserve temporal ordering
   - Compute importance scores (recency + value + relevance)

3. Import into Personal Profile:
   - Extract health preferences (supplements, fitness)
   - Extract communication style preferences
   - Extract decision-making patterns
   - Extract goal statement

4. Configuration:
   - Set TTL: All conversations = 2+ years (trading decisions time-sensitive)
   - Set retrieval weights:
     * Project-specific: weight 0.9 (highest priority)
     * Personal context: weight 0.8
     * Technical/Professional: weight 0.6
     * Creative/Research: weight 0.4
     * Transactional: weight 0.1

PHASE 4: VALIDATION & TESTING (1-2 hours)
──────────────────────────────────────────

1. Spot-check 10-15 conversations:
   - Verify embeddings capture intent correctly
   - Test retrieval accuracy (can Hestia find the conversation?)
   - Check temporal ordering is preserved

2. Run Hestia queries:
   - "When did we decide to use Coinbase for trading?"
   - "What's Andrew's preferred communication style?"
   - "What health supplements is Andrew interested in?"
   - "What was the reasoning behind the council architecture?"

3. Monitor memory system:
   - Check importance scores make sense
   - Verify retrieval latency is acceptable
   - Ensure no duplicates or conflicts

POTENTIAL RISKS & MITIGATIONS
================================================================================

RISK: Privacy exposure of personal health/financial data
MITIGATION: Import to encrypted local storage only, never cloud sync,
            audit access logs

RISK: Over-weighting older conversations (2023) vs. recent (2025)
MITIGATION: Set importance scores based on recency + relevance,
            use temporal decay function

RISK: Duplicate facts in memory system (same concept stored twice)
MITIGATION: Run deduplication pass post-import, check embeddings >0.95
            similarity

RISK: Conversation context too domain-specific to generalize
MITIGATION: Focus on extraction of principles, not just specifics.
            Hestia should understand "why" not just "what"
